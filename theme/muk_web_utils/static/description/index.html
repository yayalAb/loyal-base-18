Overview

Technical module for providing some utilities and libraries that can be used in other applications can be used in other applications. This module has no direct influence on the running system, but can be used as a dependency in other modules in order to utilise the corresponding functions.

Icon Selection Field

A selection field which offers the option of assigning a corresponding icon to the various options. If the field is used in the list view with the noLabel attribute, only one icon is displayed, otherwise the icon is displayed to the left of the selected text 

<field 
	name="company_type" 
	widget="selection_icons" 
	options="{'icons': {'person': 'user', 'company': 'building'}}"
/>

Icon Text Field

A text widget which can be used for char, text and html fields. Instead of displaying the content of the field, an icon is displayed which shows a popover with the content when clicked. This widget has only a readonly mode and is ideal for displaying text fields as information in the list view. 

<field 
	name="comment" 
	widget="text_icon" 
/>

Binary Field Preview

A preview icon has been added to the binary widget. It uses the existing file viewer to display the binary content. Because the file viewer is used, the same file types are also supported. Furthermore, the attribute noLabel is now supported.

Quick Create

If the quick create feature for many2one and many2many tags is to be deactivated by default, the system parameter muk_web_utils.disable_quick_create can be set. It is then still possible to activate the quick create feature for certain fields by explicitly passing option="{'no_quick_create': False}" to the field. 

<field 
	name="tag_ids" 
	widget="many2many_tags" 
	option="{'no_quick_create': False}"
/>

X2Many Fields

For one2many and many2many there is now a new option no_open which prevents the form view from opening even if it is a kanban view or a non-editable list view. 

<field 
	name="child_ids" 
	widget="one2many" 
	option="{'no_open': True}"
/>
